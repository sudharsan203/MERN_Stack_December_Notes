## 1. React Router

### What is React Router?
React Router is a standard library for routing in React applications. It enables navigation between different components (pages) without refreshing the browser, creating a Single Page Application (SPA) experience.

### Installation
```bash
npm install react-router-dom
```

### Key Components in React Router v6

#### 1. `createBrowserRouter`
Creates a router instance with route configuration.

**Example from our project (`routes/routes.js`):**
```javascript
import { createBrowserRouter } from 'react-router-dom';
import Layout from '../components/Layout';
import Home from '../pages/Home';
import About from '../pages/About';
import Contact from '../pages/Contact';
import NotFound from '../pages/NotFound';

const routesList = createBrowserRouter([
  {
    path: '/',
    element: <Layout />,
    errorElement: <NotFound />,
    children: [
      {
        index: true,        // Default route (/)
        element: <Home />,
      },
      {
        path: 'about',      // Route: /about
        element: <About />,
      },
      {
        path: 'contact',    // Route: /contact
        element: <Contact />,
      },
    ],
  },
]);

export default routesList;
```

#### 2. `RouterProvider`
Provides the router to your application.

**Example from our project (`index.js`):**
```javascript
import { RouterProvider } from 'react-router-dom';
import routesList from './routes/routes';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <RouterProvider router={routesList} />
  </React.StrictMode>
);
```

### Route Configuration Properties
| Property | Description |
|----------|-------------|
| `path` | URL path for the route |
| `element` | Component to render |
| `errorElement` | Component to show on errors (404) |
| `children` | Nested routes array |
| `index` | Marks as default child route |

#### 2. Functional Components with Hooks
Components that use React Hooks for state and side effects.

**Example from our project (`restApis.js`):**
```javascript
import React, { useEffect, useState } from 'react';
import axios from 'axios';

function RestApis() {
  const [listMobiles, setListMobiles] = useState([]);

  const fetchMobiles = () => {
    axios.get('http://localhost:5000/mobileslist')
      .then(response => {
        setListMobiles(response.data);

    fetchMobiles();
  }, []);

  return (
    <div>
      <h1>REST APIs</h1>
      <ul>
        {listMobiles.map((mobile, index) => (
          <li key={index}>
            {mobile.id} - {mobile.brand} - {mobile.model}
          </li>
        ))}
      </ul>
    </div>
  );
}

export default RestApis;
```

```javascript
      {/* More form fields... */}
    </form>
  );

```

## 3. Passing Data from Parent to Child Component (Props)

### What are Props?
Props (short for properties) are read-only data passed from parent to child components. They allow components to be dynamic and reusable.

### Basic Syntax

**Parent Component:**
```javascript
function ParentComponent() {
  const userName = "John";
  const userAge = 25;
  
  return (
    <ChildComponent name={userName} age={userAge} />
  );
}
```

**Child Component:**
```javascript
function ChildComponent(props) {
  return (
    <div>
      <p>Name: {props.name}</p>
      <p>Age: {props.age}</p>
    </div>
  );
}
```

### Using Destructuring (Recommended)

```javascript
// Child Component with Destructuring
function ChildComponent({ name, age }) {
  return (
    <div>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
    </div>
  );
}
```

### Passing Different Types of Data

```javascript
// Parent Component
function Parent() {
  // Different data types
  const message = "Hello World";           // String
  const count = 42;                         // Number
  const isActive = true;                    // Boolean
  const user = { name: "John", age: 25 };  // Object
  const items = ["Apple", "Banana"];        // Array
  const handleClick = () => alert("Hi");   // Function

  return (
    <Child 
      message={message}
      count={count}
      isActive={isActive}
      user={user}
      items={items}
      onClick={handleClick}
    />
  );
}

// Child Component
function Child({ message, count, isActive, user, items, onClick }) {
  return (
    <div>
      <p>{message}</p>
      <p>Count: {count}</p>
      <p>Active: {isActive ? "Yes" : "No"}</p>
      <p>User: {user.name}, {user.age}</p>
      <ul>
        {items.map((item, index) => <li key={index}>{item}</li>)}
      </ul>
      <button onClick={onClick}>Click Me</button>
    </div>
  );
}
```

### Props Rules
1. Props are **read-only** - cannot be modified by child
2. Props flow **one-way** - parent to child only
3. Props can be any JavaScript data type
4. Use **destructuring** for cleaner code

## 4. Passing Data from Child to Parent Component

### Why Pass Data from Child to Parent?
- Form submissions
- User interactions (clicks, selections)
- Updating shared state
- Communication between sibling components (via parent)

### Method: Callback Functions

The parent passes a function as a prop, and the child calls this function with data.

### Basic Pattern

**Parent Component:**
```javascript
import { useState } from 'react';
import ChildComponent from './ChildComponent';

function ParentComponent() {
  const [dataFromChild, setDataFromChild] = useState("");

  // Callback function to receive data from child
  const handleDataFromChild = (data) => {
    setDataFromChild(data);
    console.log("Received from child:", data);
  };

  return (
    <div>
      <h1>Parent Component</h1>
      <p>Data from Child: {dataFromChild}</p>
      
      {/* Pass callback function as prop */}
      <ChildComponent onSendData={handleDataFromChild} />
    </div>
  );
}
```

**Child Component:**
```javascript
function ChildComponent({ onSendData }) {
  const sendDataToParent = () => {
    const data = "Hello from Child!";
    onSendData(data);  // Call parent's function with data
  };

  return (
    <div>
      <h2>Child Component</h2>
      <button onClick={sendDataToParent}>
        Send Data to Parent
      </button>
    </div>
  );
}

export default ChildComponent;
```

### Practical Example: Search Filter

**Parent Component (ProductPage.js):**
```javascript
import { useState } from 'react';
import SearchBox from './SearchBox';
import ProductList from './ProductList';

function ProductPage() {
  const [searchTerm, setSearchTerm] = useState("");
  
  const products = [
    { id: 1, name: "iPhone 15" },
    { id: 2, name: "Samsung Galaxy" },
    { id: 3, name: "Google Pixel" },
  ];

  // Callback to receive search term from child
  const handleSearch = (term) => {
    setSearchTerm(term);
  };

  // Filter products based on search term
  const filteredProducts = products.filter(product =>
    product.name.toLowerCase().includes(searchTerm.toLowerCase())
  );

  return (
    <div>
      <h1>Product Search</h1>
      <SearchBox onSearch={handleSearch} />
      <ProductList products={filteredProducts} />
    </div>
  );
}
```

**Child Component (SearchBox.js):**
```javascript
import { useState } from 'react';

function SearchBox({ onSearch }) {
  const [inputValue, setInputValue] = useState("");

  const handleInputChange = (e) => {
    const value = e.target.value;
    setInputValue(value);
    onSearch(value);  // Send data to parent on every change
  };

  return (
    <div className="search-box">
      <input
        type="text"
        placeholder="Search products..."
        value={inputValue}
        onChange={handleInputChange}
      />
    </div>
  );
}

export default SearchBox;
```

### Practical Example: Form Submission

**Parent Component:**
```javascript
import { useState } from 'react';
import ContactForm from './ContactForm';

function ContactPage() {
  const [submissions, setSubmissions] = useState([]);

  // Callback to receive form data from child
  const handleFormSubmit = (formData) => {
    setSubmissions([...submissions, formData]);
    console.log("Form submitted:", formData);
  };

  return (
    <div>
      <h1>Contact Us</h1>
      <ContactForm onSubmit={handleFormSubmit} />
      
      <h2>Submissions ({submissions.length})</h2>
      <ul>
        {submissions.map((sub, index) => (
          <li key={index}>{sub.name} - {sub.email}</li>
        ))}
      </ul>
    </div>
  );
}
```

**Child Component (ContactForm.js):**
```javascript
import { useState } from 'react';

function ContactForm({ onSubmit }) {
  const [formData, setFormData] = useState({
    name: '',
    email: '',
    message: ''
  });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData(prev => ({ ...prev, [name]: value }));
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    onSubmit(formData);  // Send form data to parent
    setFormData({ name: '', email: '', message: '' });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="name"
        value={formData.name}
        onChange={handleChange}
        placeholder="Name"
      />
      <input
        name="email"
        value={formData.email}
        onChange={handleChange}
        placeholder="Email"
      />
      <textarea
        name="message"
        value={formData.message}
        onChange={handleChange}
        placeholder="Message"
      />
      <button type="submit">Submit</button>
    </form>
  );
}

export default ContactForm;
```

### Summary: Data Flow in React

```
┌─────────────────────────────────────────────────────┐
│                  PARENT COMPONENT                    │
│                                                      │
│   const [data, setData] = useState();               │
│                                                      │
│   const handleChildData = (childData) => {          │
│     setData(childData);                             │
│   };                                                │
│                                                      │
│   ┌──────────────┐     ┌──────────────────────┐    │
│   │ Props (↓)    │     │ Callback Function (↑) │    │
│   │ data={data}  │     │ onData={handleChild}  │    │
│   └──────┬───────┘     └──────────┬───────────┘    │
│          │                        │                  │
└──────────┼────────────────────────┼──────────────────┘
           │                        │
           ▼                        │
┌──────────────────────────────────────────────────────┐
│                   CHILD COMPONENT                     │
│                                                       │
│   function Child({ data, onData }) {                 │
│     // Use data from parent                          │
│     // Call onData(newData) to send data to parent   │
│   }                                                  │
│                                                       │
└──────────────────────────────────────────────────────┘
```

### Key Points
| Direction | Method | Example |
|-----------|--------|---------|
| Parent → Child | Props | `<Child name="John" />` |
| Child → Parent | Callback Functions | `<Child onData={handleData} />` |

## Quick Reference

### React Router Setup Steps
1. Install: `npm install react-router-dom`
2. Create routes configuration with `createBrowserRouter`
3. Wrap app with `RouterProvider`
4. Use `Outlet` in layout for child routes
5. Use `NavLink` or `Link` for navigation

### Component Types
- **Functional Components** - Functions that return JSX
- **Layout Components** - Provide structure (header, footer, sidebar)
- **Page Components** - Represent full pages/views
- **Stateful Components** - Manage local state with `useState`

### Data Flow
- **Parent to Child**: Pass via props
- **Child to Parent**: Pass callback function as prop, child calls it with data
