## HTML Structure

### Basic Document Structure
```html
<!DOCTYPE html>
<html>
    <head>
        <!-- Meta information, scripts, styles -->
    </head>
    <body>
        <!-- Visible content -->
    </body>
</html>
```

**Components:**
- `<!DOCTYPE html>` - Declares HTML5 document type
- `<html>` - Root element
- `<head>` - Contains metadata, scripts, and styles
- `<body>` - Contains visible page content

### Head Section Elements
```html
<meta charset="utf-8">
<title>My Sample Page</title>
```
- **meta charset** - Defines character encoding (UTF-8 supports most characters)
- **title** - Page title shown in browser tab

## CSS Styling Methods

Our sample demonstrates **THREE** ways to apply CSS:

### 1. External CSS (Commented Out)
```html
<!-- <link rel="stylesheet" href="./sample.css"> -->
```
- **Purpose:** Link external CSS file
- **Advantage:** Reusable across multiple pages
- **File:** `sample.css`

**sample.css content:**
```css
h1, h2, h3, h4, h5, h6 {
    color: red;
    text-decoration: underline;
}
```
- Targets all heading elements
- Sets color to red and adds underline

### 2. Internal CSS (Style Tag)
```html
<style>
    #headingId {
        color: blue;
        text-decoration: none;
    }
    .heading-style {
        color: red;
        text-decoration: underline;
    }
</style>
```
- **Purpose:** Define styles within the HTML document
- **Scope:** Applies only to current page
- **Location:** Inside `<head>` section

### 3. Inline CSS (Style Attribute)
```html
<h1 class="heading-style" style="color: green;">Welcome to My JS Page</h1>
```
- **Purpose:** Apply styles directly to an element
- **Highest Specificity:** Overrides external and internal styles
- **Use Case:** Quick, element-specific styling

---

## CSS Specificity

**Specificity hierarchy** (from lowest to highest):
1. **Element selectors** - `h1`, `table`, `td`
2. **Class selectors** - `.heading-style`, `.div-container`
3. **ID selectors** - `#headingId`, `#idStyle`
4. **Inline styles** - `style="color: green;"`

### Example from Code:
```html
<h4 id="headingId" class="heading-style" style="color: green;text-transform: uppercase;">
    JS is working!
</h4>
```

**Style Resolution:**
- External CSS would make it red (if uncommented)
- Class `.heading-style` tries to make it red with underline
- ID `#headingId` overrides class, making it blue
- **Inline style WINS** - Final color is **green** with uppercase transformation

## JavaScript Integration

Our sample demonstrates **THREE** ways to include JavaScript:

### 1. External JavaScript File
```html
<script src="./sample.js"></script>
```
- Links to external `sample.js` file
- **Content:** `console.log("Hello, CoffeeScript!");`
- **Advantage:** Code reusability and organization

### 2. Internal JavaScript (Script Tag)
```html
<script>
    console.log("This is a sample HTML page with js integration.");
</script>
```
- JavaScript code written directly in HTML
- Executes when page loads

### 3. Inline JavaScript (Event Handlers)
```html
<h2 class="heading-style" onClick="console.log('onclick Method')">
    Inline Script This is a simple JS example
</h2>
```
- JavaScript in HTML attributes
- Executes on specific events (click, hover, etc.)
- **Event:** `onClick` triggers when element is clicked

## HTML Elements Covered

### Headings (h1-h6)
```html
<h1>Welcome to My JS Page</h1>
<h2>This is a simple JS example</h2>
<h3>Check the console for output</h3>
<!-- ... h4, h5, h6 -->
```
- Six levels of headings
- `<h1>` is largest, `<h6>` is smallest
- Used for document structure and SEO

### Division Container
```html
<div>
    <h1 class="div-container">This is a div container</h1>
</div>
```
- **Purpose:** Group elements for styling/layout
- Block-level element
- Used with CSS classes for styling

### Table
```html
<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>Skill</th>
    </tr>
    <tr>
        <td>Javascript</td>
        <td>25</td>
        <td>Programming</td>
    </tr>
</table>
```

**Table Elements:**
- `<table>` - Table container
- `<tr>` - Table row
- `<th>` - Table header cell (bold by default)
- `<td>` - Table data cell

**CSS for Table:**
```css
table, th, td {
    border: 1px solid black;
    border-collapse: collapse;
    padding: 5px;
}
```
- Adds borders to all cells
- `border-collapse` removes spacing between borders

### Form Elements
```html
<form action=""></form>
<button>Click Me</button>
```
- `<form>` - Container for user input elements
- `action` attribute (empty here) would specify where to submit data

### Other Elements
```html
<img src="" alt="">          <!-- Image -->
<a href=""></a>              <!-- Hyperlink -->
<p>Hi Paragraph</p>          <!-- Paragraph -->
```

## Key Concepts Summary

### 1. CSS Box Model
```css
.div-container {
    border: 2px solid blue;
    padding: 10px;
}
```
- **border** - Outline around element
- **padding** - Space inside border, around content

### 2. CSS Properties Used

| Property | Purpose | Example |
|----------|---------|---------|
| `color` | Text color | `color: red;` |
| `text-decoration` | Text styling | `text-decoration: underline;` |
| `border` | Element border | `border: 1px solid black;` |
| `padding` | Inner spacing | `padding: 10px;` |
| `font-style` | Text style | `font-style: italic;` |
| `text-transform` | Text case | `text-transform: uppercase;` |
| `border-collapse` | Table borders | `border-collapse: collapse;` |

### 3. Multiple Classes on One Element
```html
<h3 class="heading-style" id="idStyle">Check the console for output</h3>
```
- Element can have both class and id
- Styles from both are applied
- ID has higher specificity than class

### 4. Console Output
All three JavaScript methods output to browser console:
1. External JS: `"Hello, CoffeeScript!"`
2. Internal JS: `"This is a sample HTML page with js integration."`
3. Inline JS: `"onclick Method"` (when h2 is clicked)

**To view:** Open browser Developer Tools (F12) → Console tab

## Best Practices Learned

✅ **DO:**
- Use external CSS for large projects
- Use meaningful class and id names
- Comment out code instead of deleting (for reference)
- Structure HTML with proper indentation

❌ **AVOID:**
- Overusing inline styles (hard to maintain)
- Too many inline event handlers (use addEventListener instead)
- Mixing too many styling methods (choose one approach)
